\chapter{Glossary and Decision Tables}

\section*{Selected terms}

\begin{description}
\item[Binding] A lifecycle-owned connection between a model and a Swing component or component property.
\item[Behavior] A named reusable Swing component capability, such as validation decoration, command binding, or help-on-F1.
\item[Descriptor] Immutable metadata describing a field, table, column, command, resource, or help topic before Swing components are created.
\item[EDT] The AWT/Swing event dispatch thread; the normal owner of Swing component state.
\item[MigLayout] The preferred layout manager for nontrivial book examples and ordinary Corusco Swing screens.
\item[Presentation model] Screen-owned state such as field text, selected row, dirty flag, enabled command state, or validation problems.
\item[Problem] Structured validation or parsing feedback with severity, code, source, target, and message.
\item[Scope] An object that owns cleanup for subscriptions, listeners, bindings, behaviors, tasks, or detachable state.
\end{description}
