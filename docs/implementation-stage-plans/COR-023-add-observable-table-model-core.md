# COR-023 Add Observable Table Model Core

Commit message:

```text
COR-023 Add observable table model core
```

## Roadmap Coverage

Roadmap Stage 12: Table Model and Table Column Descriptors, foundational
typed table slice.

## Objective

Add handwritten table key, column, descriptor, and Swing table-model APIs that
can later be generated by the annotation processor. This stage should make
read-only and editable record-row tables practical without JavaBeans property
paths or arbitrary public column-name strings.

## Dependencies

- Requires COR-004 typed key conventions.
- Requires COR-017 observable list core.
- Requires COR-022 EDT dispatcher when background-originated row changes need
  to reach Swing models.

## Scope

Add core table APIs:

- `TableKey<R>`
- `ColumnKey<R, V>`
- `ColumnDefaults`
- `ColumnCapabilities`
- `ColumnDescriptor<R, V>`
- `Column<R, V>`
- `TableDescriptor<R>`

Add Swing table API:

- `ObservableTableModel<R>`

Required behavior:

- table and column keys use stable ids and row/value type metadata;
- columns expose value extraction without reflection or string property paths;
- editable columns use an explicit row updater/wither function;
- table descriptors preserve ordered columns immutably;
- Swing table model reads row data from `ObservableList<R>`;
- observable-list insert/delete/update changes map to Swing table row events;
- table cell edits update the source row through the column updater;
- public Javadocs document ownership, threading, mutability, nullability, and
  generated-code expectations.

## Required Deliverables

- Production table APIs in `corusco-core` and `corusco-swing`.
- Unit tests for key validation, descriptors, editable/read-only columns, row
  event translation, and source updates from table edits.
- Example showing a typed editable record-row table with method-body comments
  explaining generated-style keys, immutable row updates, and EDT ownership.
- Stage-plan index update.

## Out of Scope

- Annotation generation of table columns.
- Table selection binding.
- Table cell problem targets/decorations beyond preserving a future key shape.
- Persistent column state.
- Resource bundle lookup for header text.

## Implementation Steps

1. Add this stage plan and index entry.
2. Add core table key/descriptor/column classes.
3. Add Swing `ObservableTableModel`.
4. Add focused core and Swing tests.
5. Add a table-model example and smoke test.
6. Run AudEnv compact test/build recommendations and review scans.

## Acceptance Checks

- User API references typed `ColumnKey`/`Column` objects, not property-name
  strings.
- `ObservableTableModel` fires precise insert/delete/update events for source
  list changes.
- Editable record rows can be replaced through an explicit updater function.
